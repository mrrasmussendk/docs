Vitruvian is a .NET GOAP agent runtime that breaks user requests into dependency-aware plans, runs independent steps in parallel, and gates side-effecting actions behind human approval.

## Quickstart

<Info>
  **Prerequisites**:
  - [.NET 8 SDK](https://dotnet.microsoft.com/en-us/download)
  - [Git](https://git-scm.com/)
  - An API key for OpenAI, Anthropic, or Gemini
</Info>

```bash
git clone https://github.com/mrrasmussendk/Vitruvian.git
cd Vitruvian
dotnet build Vitruvian.sln
dotnet test Vitruvian.sln
./scripts/install.sh
dotnet run --project src/Vitruvian.Cli
```

Try requests like:

```text
> What is the weather tomorrow?
> Create a file called notes.txt with content "Hello World"
> Read notes.txt then summarize it
```

## Key features

- **GOAP planning**: decomposes requests into execution steps with dependency edges before running.
- **Parallel execution**: runs independent steps concurrently.
- **Human-in-the-loop (HITL)**: requires approval for write, delete, and execute operations.
- **Result caching**: reuses output for identical module and input pairs.
- **Compound requests**: splits multi-intent prompts and routes each task through the same pipeline.
- **Module extensibility**: lets you implement `IVitruvianModule` and register via DI or plugin DLL.

## Request lifecycle

```text
User Request
    ↓
PLAN (GoapPlanner creates an ExecutionPlan)
    ↓
EXECUTE (cache check → approval gate → module execution)
    ↓
MEMORY (store result + conversation turn)
    ↓
Response
```
